# Deployment Architecture

## 1. Frontend (Vercel)
The Next.js 15 frontend is deployed seamlessly to Vercel. Vercel automatically handles Next.js specific optimizations (Image optimization, edge caching, serverless functions).
- **CI/CD**: Connected directly to the main GitHub branch.

## 2. Backend (Railway/Render)
The NestJS backend is deployed as a Docker container on a PaaS provider (like Railway or Render).
- **CI/CD**: Deploys are triggered via GitHub Actions upon merge to the main branch.

## 3. Database (Supabase)
Supabase handles the PostgreSQL database, Authentication, and Storage. Environment variables connect the frontend and backend to the Supabase instance.

## 4. Environments
- **Production**: Connected to `main` branch.
- **Preview/Staging**: Automatically generated by Vercel on Pull Requests.
