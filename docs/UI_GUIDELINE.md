# UI Guidelines

This document outlines the design system, aesthetics, colors, typography, and responsive design principles for the IT Developer Portfolio.

## 1. Design Principles
- **Modern & Premium**: Use dark mode as the primary theme. Glassmorphism, deep shadows, and vibrant accent colors to create a premium feel.
- **Dynamic**: Interfaces must feel alive. Use micro-interactions, hover states, and smooth transitions.
- **Accessible**: High contrast ratios, accessible focus states, and semantic HTML structure.

## 2. Typography
- **Primary Font**: `Inter` or `Geist` (sans-serif) for body text and UI elements.
- **Display Font**: `Outfit` or `Cal Sans` for bold headings.
- **Monospace Font**: `Fira Code` or `JetBrains Mono` for code snippets.

## 3. Color Palette
- **Background**: Deep rich dark (e.g., `#0a0a0a`)
- **Surface**: Elevated dark panels (e.g., `#141414` with semi-transparency)
- **Primary Accent**: Vibrant electric blue or purple (e.g., `#3b82f6` to `#8b5cf6` gradients)
- **Text Primary**: Off-white (e.g., `#ededed`)
- **Text Secondary**: Muted gray (e.g., `#a1a1aa`)

## 4. Components & Layout
- **Spacing**: Use an 8px base grid (e.g., 8px, 16px, 24px, 32px, 48px).
- **Border Radius**: Use rounded corners (typically `0.5rem` to `1rem`) for a soft, modern look.
- **Responsive**: Mobile-first design. Ensure seamless scaling up to 4k displays. Breakpoints at `sm` (640px), `md` (768px), `lg` (1024px), `xl` (1280px).

## 5. Animations
- **Library**: Framer Motion.
- **Timing**: Snappy but smooth. Duration `0.2s` - `0.4s` for UI transitions. Spring animations for playful interactions.
- **Scroll Effects**: Fade-up on scroll for sections to introduce content gracefully.
