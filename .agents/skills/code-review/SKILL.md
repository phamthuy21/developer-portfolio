# Code Review Skill

1. Ensure code adheres to `CODING_STANDARD.md`.
2. Look for hardcoded strings or secrets (they should be env vars).
3. Verify type safety (no `any`).
4. Ensure components are reusable and not overly complex.
5. Suggest refactors for better readability (DRY, KISS).
